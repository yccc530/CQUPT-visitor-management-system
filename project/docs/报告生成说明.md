# 报告生成说明

## 一、功能说明

`scripts/generate_report.py` 用于自动生成数据库原理课程设计报告。脚本会读取 `docs/report_parts/`、`docs/diagrams.md`、`database/schema.sql`、`database/query_examples.sql`、`screenshots/manifest.json`、`screenshots/*.png`、`docs/API接口说明.md`、`docs/系统测试说明.md` 和 `docs/测试用例.md`，并整合为 Markdown 报告和 Word 报告。

## 二、生成文件

| 文件 | 说明 |
|---|---|
| `docs/重庆邮电大学访客管理系统设计报告.md` | 课程设计报告 Markdown 正文。 |
| `docs/重庆邮电大学访客管理系统设计报告.docx` | 课程设计报告 Word 文档。 |

## 三、运行方法

```bash
cd project
python scripts/generate_report.py
```

## 四、依赖说明

Word 导出依赖 `python-docx`：

```bash
python -m pip install python-docx
```

如果未安装该依赖，脚本仍会生成 Markdown 报告，并输出 Word 导出失败说明。

## 五、素材要求

生成报告前建议先执行：

```bash
bash scripts/init_database.sh
bash scripts/run_all.sh
```

这样可以确保数据库演示数据、运行截图和 `screenshots/manifest.json` 都是最新状态。

## 六、质量检查

生成后请重点检查：

1. 报告章节是否包含系统定义、需求分析、系统设计、详细设计、系统实现与测试、总结。
2. Mermaid 图是否能够在 Markdown 预览器中渲染。
3. SQL 表名、字段名和查询语句是否与 `database/` 目录一致。
4. 截图是否能正常打开，Word 中是否插入代表性页面截图。
5. 测试用例和 Bug 修复日志是否与实际验证结果一致。
