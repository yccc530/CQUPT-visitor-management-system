# 运行说明

## 一、运行前准备

请确认本机已安装或可用以下环境：

| 工具 | 建议版本 | 用途 |
|---|---|---|
| MySQL | 8.x | 存储系统业务数据 |
| JDK | 17 | 运行 Spring Boot 后端 |
| Maven | 3.9+ | 构建后端项目 |
| Node.js/npm | Node.js 18+ 或更高 | 启动 Vue 前端 |
| Python | 3.10+ | 自动生成课程设计报告 |
| Chrome | 最新版 | Playwright 自动截图兜底浏览器 |

当前项目已经验证可在 Windows 11 环境下运行。若使用 Git Bash，可直接执行 `scripts/*.sh`。

## 二、数据库初始化

```bash
cd project
bash scripts/init_database.sh
```

默认连接参数：

| 参数 | 默认值 |
|---|---|
| DB_HOST | localhost |
| DB_PORT | 3306 |
| DB_USERNAME | root |
| DB_PASSWORD | root |

自定义连接示例：

```bash
DB_USERNAME=root DB_PASSWORD=你的密码 bash scripts/init_database.sh
```

## 三、启动后端

```bash
cd project
bash scripts/run_backend.sh
```

启动成功后访问：

```text
http://127.0.0.1:8080/swagger-ui.html
```

## 四、启动前端

另开终端执行：

```bash
cd project
bash scripts/run_frontend.sh
```

访问：

```text
http://127.0.0.1:5173/login
```

## 五、一键截图验证

如果只想自动启动服务、截图并退出，可执行：

```bash
cd project
bash scripts/run_all.sh
```

脚本会：

1. 检查 8080 和 5173 是否空闲。
2. 启动后端服务。
3. 启动前端服务。
4. 使用 Playwright 登录不同角色。
5. 生成 19 张截图。
6. 写入 `screenshots/manifest.json`。
7. 自动清理 8080 和 5173 服务进程。

## 八、验证命令

```bash
cd project
bash scripts/init_database.sh
mvn -f backend/pom.xml -DskipTests package
cd frontend && npm run build && cd ..
bash scripts/run_all.sh
python scripts/generate_report.py
```

若上述命令均成功，说明数据库、后端、前端、自动截图和报告生成链路可用。

## 九、注意事项

- `scripts/init_database.sh` 会重建 `cqupt_visitor_system` 数据库，请勿在生产数据上执行。
- `project/.tools/`、`backend/target/`、`frontend/node_modules/`、`frontend/dist/` 和 `logs/` 属于本地构建或缓存目录，不作为提交内容。
- 当前前端构建存在 chunk 体积提示，不影响课程设计演示和运行。
- 如果 MySQL 未注册为 Windows 服务，重启电脑后需要重新启动 MySQL。
