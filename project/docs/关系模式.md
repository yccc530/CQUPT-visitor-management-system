# 关系模式

## 1. 关系模式说明

关系模式采用如下记法：

`表名(主键, 普通字段, 外键字段 -> 被引用表.字段)`

主键统一为 `id`。外键字段在关系模式中使用 `->` 标明引用关系。所有表在物理实现时默认包含 `create_time`、`update_time`，需要保留历史和软删除能力的表包含 `deleted`。

## 2. 系统权限与组织关系模式

1. `department(id, parent_id -> department.id, dept_code, dept_name, leader_user_id -> sys_user.id, phone, sort_order, status, create_time, update_time, deleted)`
2. `sys_user(id, username, password_hash, real_name, phone, email, department_id -> department.id, user_type, status, last_login_time, create_time, update_time, deleted)`
3. `sys_role(id, role_code, role_name, role_desc, status, create_time, update_time, deleted)`
4. `sys_permission(id, permission_code, permission_name, permission_type, parent_id -> sys_permission.id, route_path, component_path, api_path, sort_order, status, create_time, update_time, deleted)`
5. `sys_user_role(id, user_id -> sys_user.id, role_id -> sys_role.id, create_time, update_time, deleted)`
6. `sys_role_permission(id, role_id -> sys_role.id, permission_id -> sys_permission.id, create_time, update_time, deleted)`

## 3. 访客预约关系模式

1. `visitor(id, visitor_name, id_type, id_number, phone, gender, unit_name, visitor_type, remark, create_time, update_time, deleted)`
2. `visitor_vehicle(id, visitor_id -> visitor.id, plate_number, vehicle_type, vehicle_color, brand, status, create_time, update_time, deleted)`
3. `visit_apply(id, apply_no, visitor_id -> visitor.id, host_user_id -> sys_user.id, department_id -> department.id, vehicle_id -> visitor_vehicle.id, visit_reason, plan_start_time, plan_end_time, apply_status, access_status, companion_count, submit_time, cancel_time, cancel_reason, remark, create_time, update_time, deleted)`
4. `visitor_companion(id, apply_id -> visit_apply.id, companion_name, id_type, id_number, phone, relation_remark, create_time, update_time, deleted)`

## 4. 审批、通行与出入校关系模式

1. `approval_record(id, apply_id -> visit_apply.id, approval_step, approver_user_id -> sys_user.id, approval_result, approval_comment, approval_time, sort_order, create_time, update_time, deleted)`
2. `pass_code(id, apply_id -> visit_apply.id, pass_code, qr_content, valid_from, valid_to, pass_status, used_time, verify_count, create_time, update_time, deleted)`
3. `access_record(id, apply_id -> visit_apply.id, visitor_id -> visitor.id, pass_code_id -> pass_code.id, entry_gate_id -> campus_gate.id, exit_gate_id -> campus_gate.id, entry_guard_id -> sys_user.id, exit_guard_id -> sys_user.id, entry_time, exit_time, access_status, overtime_flag, abnormal_reason, create_time, update_time, deleted)`
4. `campus_gate(id, gate_code, gate_name, gate_location, gate_type, status, remark, create_time, update_time, deleted)`

## 5. 风险控制、通知和日志关系模式

1. `blacklist(id, visitor_id -> visitor.id, id_number, phone, reason, level, start_time, end_time, status, operator_user_id -> sys_user.id, create_time, update_time, deleted)`
2. `notice(id, receiver_user_id -> sys_user.id, receiver_type, title, content, business_type, business_id, read_status, read_time, create_time, update_time, deleted)`
3. `operation_log(id, operator_user_id -> sys_user.id, operator_name, module_name, operation_type, request_method, request_url, operation_result, ip_address, operation_time, error_message, create_time, update_time, deleted)`

## 6. 字典与自动化材料关系模式

1. `dict_type(id, dict_code, dict_name, description, status, create_time, update_time, deleted)`
2. `dict_item(id, dict_type_id -> dict_type.id, item_code, item_label, item_value, sort_order, status, remark, create_time, update_time, deleted)`
3. `screenshot_record(id, screenshot_code, page_name, route_path, role_code, file_path, description, capture_time, status, created_by -> sys_user.id, create_time, update_time, deleted)`
4. `report_record(id, report_code, report_name, markdown_path, word_path, generate_status, generate_time, generated_by -> sys_user.id, error_message, create_time, update_time, deleted)`

## 7. 主要联系映射

| 概念联系 | 关系模式实现 |
|---|---|
| 一个部门包含多个系统用户 | `sys_user.department_id -> department.id` |
| 一个用户可以拥有多个角色 | `sys_user_role.user_id -> sys_user.id`, `sys_user_role.role_id -> sys_role.id` |
| 一个角色可以拥有多个权限 | `sys_role_permission.role_id -> sys_role.id`, `sys_role_permission.permission_id -> sys_permission.id` |
| 一个访客可以提交多个预约申请 | `visit_apply.visitor_id -> visitor.id` |
| 一个预约申请可以包含多个随行人员 | `visitor_companion.apply_id -> visit_apply.id` |
| 一个预约申请可以关联车辆信息 | `visit_apply.vehicle_id -> visitor_vehicle.id` |
| 一个预约申请可以产生多条审批记录 | `approval_record.apply_id -> visit_apply.id` |
| 审批通过预约可以生成一个通行凭证 | `pass_code.apply_id -> visit_apply.id`，后续设置唯一约束 |
| 一个预约申请可以产生出入校记录 | `access_record.apply_id -> visit_apply.id` |
| 一个校门可以关联多条出入校记录 | `access_record.entry_gate_id -> campus_gate.id`, `access_record.exit_gate_id -> campus_gate.id` |
| 一个访客可能被加入黑名单 | `blacklist.visitor_id -> visitor.id` |
| 一个用户可以产生多条操作日志 | `operation_log.operator_user_id -> sys_user.id` |
