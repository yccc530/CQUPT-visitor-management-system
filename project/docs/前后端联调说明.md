# 前后端联调说明

## 一、联调目标

本阶段围绕“重庆邮电大学智慧访客预约与出入校管理系统”进行数据库、后端和前端的一致性检查，重点解决前端页面字段、后端接口返回、数据库表字段和统计报表接口之间的衔接问题。

## 二、本阶段已完成修复

| 问题 | 修复方式 | 涉及位置 |
|---|---|---|
| 前端预约列表需要 `visitorName`、`phone`、`departmentName`，后端原来只返回 ID | 后端新增展示字段补全服务，接口返回时自动根据外键补全访客、被访人、部门、车辆信息 | `IntegrationViewService`、`VisitApplyController`、`VisitorWorkflowController` |
| 当前在校、超时未离校页面需要 `applyNo`、`visitorName`、`entryGateName` | 后端为 `access_record` 返回补充预约编号、访客姓名、手机号、校门名、安保人员名、通行码 | `AccessRecord`、`AccessRecordController`、`VisitorWorkflowController` |
| 黑名单页面需要访客姓名和操作人姓名 | 后端为 `blacklist` 返回补充 `visitorName`、`operatorName` | `Blacklist`、`BlacklistController` |
| 用户管理页面需要部门名称且不应暴露密码摘要 | 后端为 `sys_user` 返回补充 `departmentName`，并在返回前清空 `passwordHash` | `SysUser`、`SysUserController` |
| 统计报表页面原来主要使用静态数据 | 新增 `/api/statistics/dashboard` 等统计接口，前端统计页优先读取真实接口 | `StatisticsController`、`frontend/src/api/statistics.js`、`Statistics.vue` |
| 前端搜索条件传给后端但后端不处理 | 为预约、出入校、黑名单、用户、日志列表补充常用筛选条件 | 多个 Controller |
| 部门待审批 SQL 与状态流转不一致 | 将 `PENDING_DEPT` 单状态查询修正为 `HOST_CONFIRMED` 和 `PENDING_DEPT` | `query_examples.sql`、`08_SQL查询语句和说明.md` |
