# 权限设计

项目名称：重庆邮电大学智慧访客预约与出入校管理系统

## 1. 设计目标

权限系统采用“登录认证 + JWT Token + RBAC 角色权限控制”的方式实现。用户登录成功后，后端签发 JWT Token；前端后续请求在 `Authorization` 请求头中携带 `Bearer Token`。后端通过过滤器解析 Token，加载当前用户、角色和权限，并由 Spring Security 对接口进行统一鉴权。

## 2. 认证流程

1. 用户调用 `POST /api/auth/login` 提交用户名和密码。
2. 后端根据 `sys_user.username` 查询账号，并使用 BCrypt 校验 `password_hash`。
3. 校验通过后，后端查询 `sys_user_role`、`sys_role`、`sys_role_permission`、`sys_permission`，组装当前用户角色和权限。
4. 后端生成 JWT Token，返回 Token、过期时间和用户信息。
5. 前端将 Token 放入请求头：`Authorization: Bearer <token>`。
6. 后端 `JwtAuthenticationFilter` 对受保护接口进行身份解析。
7. 退出登录时调用 `POST /api/auth/logout`，后端将当前 Token 加入内存撤销表，当前进程内该 Token 立即失效。

## 4. 角色权限矩阵

| 角色编码 | 角色名称 | 菜单权限 | 接口权限 | 数据范围 |
|---|---|---|---|---|
| `ADMIN` | 系统管理员 | 全部菜单 | 全部后端接口 | 全部数据 |
| `HOST` | 被访人 | 首页、待确认预约、访客记录查询 | 查询本人作为被访人的预约、查看/新增审批记录 | `visit_apply.host_user_id = 当前用户ID` |
| `DEPT_APPROVER` | 部门审批人员 | 首页、部门审批、访客记录查询、统计报表 | 查询本部门预约、查看/新增审批记录 | `visit_apply.department_id = 当前用户部门ID` |
| `GATE_GUARD` | 门岗安保人员 | 首页、门岗核验、入校登记、离校登记、当前在校、超时未离校 | 访问通行凭证和出入校记录相关接口 | 门岗通行数据 |
| `SCHOOL_MANAGER` | 校级管理人员 | 首页、当前在校、超时未离校、访客记录查询、统计报表 | 只读查看访客、预约、审批、出入校统计数据 | 全校只读数据 |
| `VISITOR` | 访客 | 访客预约申请、我的预约 | 提交预约、查看本人预约状态和通行凭证 | 本人记录，后续访客端扩展时绑定访客身份 |

## 5. 主要接口保护规则

| 接口路径 | 允许角色 | 说明 |
|---|---|---|
| `POST /api/auth/login` | 匿名 | 登录并获取 Token |
| `POST /api/auth/logout` | 已登录用户 | 退出登录并撤销 Token |
| `GET /api/auth/me` | 已登录用户 | 获取当前用户信息 |
| `GET /api/auth/menus` | 已登录用户 | 获取当前用户菜单权限 |
| `/api/sys-users/**` | `ADMIN` | 用户管理 |
| `/api/sys-roles/**` | `ADMIN` | 角色管理 |
| `/api/sys-permissions/**` | `ADMIN` | 权限管理 |
| `/api/departments/**` | `ADMIN` | 部门管理 |
| `/api/campus-gates/**` | `ADMIN` | 校门配置管理 |
| `GET /api/visitors/**` | `ADMIN`, `SCHOOL_MANAGER` | 访客档案查看 |
| `POST /api/visitors/**` | `ADMIN`, `VISITOR` | 新增访客档案 |
| `GET /api/visit-applies/**` | `ADMIN`, `SCHOOL_MANAGER`, `HOST`, `DEPT_APPROVER` | 预约记录查询，后端附加数据范围过滤 |
| `POST /api/visit-applies/**` | `ADMIN`, `VISITOR` | 提交预约申请 |
| `GET /api/approval-records/**` | `ADMIN`, `SCHOOL_MANAGER`, `HOST`, `DEPT_APPROVER` | 审批记录查询 |
| `POST /api/approval-records/**` | `ADMIN`, `HOST`, `DEPT_APPROVER` | 新增确认或审批记录 |
| `GET /api/pass-codes/**` | `ADMIN`, `GATE_GUARD`, `VISITOR` | 查询通行凭证 |
| `GET /api/access-records/**` | `ADMIN`, `SCHOOL_MANAGER`, `GATE_GUARD` | 查询出入校记录 |
| `POST /api/access-records/**` | `ADMIN`, `GATE_GUARD` | 新增入校或离校记录 |
| `/api/blacklists/**` | `ADMIN` | 黑名单管理 |
| `/api/operation-logs/**` | `ADMIN` | 系统日志管理 |

## 6. 无权限响应

未登录或 Token 无效时返回：

```json
{
  "code": 401,
  "message": "未登录或 Token 无效",
  "data": null,
  "success": false
}
```

角色无权访问接口时返回：

```json
{
  "code": 403,
  "message": "没有权限访问该接口",
  "data": null,
  "success": false
}
```

## 7. 登录测试方式

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"admin\",\"password\":\"123456\"}"
```

登录成功后取返回的 `token`，访问受保护接口：

```bash
curl http://localhost:8080/api/auth/me \
  -H "Authorization: Bearer <token>"
```

获取菜单权限：

```bash
curl http://localhost:8080/api/auth/menus \
  -H "Authorization: Bearer <token>"
```

退出登录：

```bash
curl -X POST http://localhost:8080/api/auth/logout \
  -H "Authorization: Bearer <token>"
```
