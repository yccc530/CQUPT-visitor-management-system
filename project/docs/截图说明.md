# 截图说明

本项目使用 Playwright 自动访问前端页面并生成课程设计报告所需运行截图。截图脚本位于 `scripts/capture_screenshots.js`，默认截图分辨率为 `1440x900`，截图文件统一保存到 `screenshots/`，同时同步复制到 `report-latex/figures/`，供 LaTeX 报告第五章“系统实现与测试”引用。

## 截图前置条件

1. MySQL 服务已启动，数据库账号与 `scripts/init_database.sh` 中配置一致。
2. 后端服务已启动，默认地址为 `http://127.0.0.1:8080`。
3. 前端服务已启动，默认地址为 `http://127.0.0.1:5173`。
4. 已安装 Node.js、npm 与 Playwright 依赖。
5. 建议先导入 `database/demo_seed.sql` 或执行 `scripts/init_database.sh`，保证页面表格和统计图有足够演示数据。

## 运行命令

在 `project/` 目录下执行：

```bash
node scripts/capture_screenshots.js
```

也可以使用一键脚本启动后端、前端、截图和报告生成：

```bash
bash scripts/run_all.sh
```

如需跳过截图前数据库初始化，可执行：

```bash
INIT_DATABASE_BEFORE_SCREENSHOTS=false node scripts/capture_screenshots.js
```

## manifest.json 字段

`screenshots/manifest.json` 中每条记录包含：

| 字段 | 说明 |
|---|---|
| id | 截图业务标识，例如 `dashboard` |
| title | 截图中文标题 |
| file | 截图在项目中的路径，例如 `screenshots/02_dashboard.png` |
| latexFile | LaTeX 报告引用路径，例如 `figures/02_dashboard.png` |
| chapter | 对应报告章节 |
| description | 截图说明文字 |

## 与 LaTeX 报告联动

截图脚本会将 PNG 文件复制到 `report-latex/figures/`，并生成 `report-latex/figures/screenshot_manifest.json`。`scripts/generate_latex_report.py` 会读取 `screenshots/manifest.json`，在 `report-latex/chapters/05_implementation_testing.tex` 中生成对应的 `figure` 环境：

```tex
\begin{figure}[H]
  \centering
  \includegraphics[width=0.92\textwidth]{figures/02_dashboard.png}
  \caption{首页统计驾驶舱}
\end{figure}
```

## 质量检查

- 表格页面至少应显示多条演示数据。
- 首页和统计报表页的折线图、柱状图、饼图不应为空。
- 门岗核验页应能展示核验输入区和结果区。
- 截图文件名应保持固定，便于报告稳定引用。
- `manifest.json` 的 `file` 与 `latexFile` 路径必须能实际找到文件。
