# 截图说明

## 一、截图目标

自动截图用于课程设计报告“系统实现与测试”章节，展示系统主界面、业务模块界面、查询页和报表页的实际运行效果。

## 二、截图脚本

脚本位置：

```text
scripts/capture_screenshots.js
```

脚本会完成以下操作：

1. 检查前端服务 `http://127.0.0.1:5173/login` 是否可访问。
2. 检查后端服务 `http://127.0.0.1:8080/swagger-ui.html` 是否可访问。
3. 使用 Playwright 打开 Chromium 浏览器。
4. 按不同角色登录系统。
5. 访问 19 个核心页面。
6. 将截图保存到 `screenshots/`。
7. 生成 `screenshots/manifest.json`。

## 四、运行方式

先确保数据库、后端和前端已经启动，然后执行：

```bash
cd project
node scripts/capture_screenshots.js
```

如果未安装 Playwright，请执行：

```bash
cd project/frontend
npm install -D playwright
npx playwright install chromium
```

如果 Chromium 下载较慢或失败，脚本会自动检测系统 Chrome。也可以通过 `CHROME_PATH` 指定浏览器路径，例如 `C:\Program Files\Google\Chrome\Application\chrome.exe`。

也可以在 Git Bash 中使用一键脚本：

```bash
cd project
bash scripts/run_all.sh
```

## 五、manifest 说明

`manifest.json` 中每条记录包含：截图编号、页面名称、路由路径、角色、文件路径、说明文字、报告章节、截图时间和状态。后续报告生成脚本可以读取该文件，将截图自动插入课程设计报告。
