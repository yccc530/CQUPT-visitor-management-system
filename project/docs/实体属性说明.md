# 实体属性说明

## 1. 属性设计原则

1. 每个实体都设置稳定主键 `id`，后续逻辑结构设计统一转换为 `BIGINT` 类型。
2. 业务编号如预约编号、角色编码、权限编码、校门编码等作为候选键或唯一约束使用。
3. 时间类属性在逻辑设计中统一扩展为 `create_time`、`update_time`，业务时间保留独立语义，如 `plan_start_time`、`entry_time`、`exit_time`。
4. 状态类属性使用清晰的业务状态名称，并在后续字典表中维护可选值。
5. 多对多联系在概念设计中作为联系存在，在逻辑设计中转换为关联表。

## 2. 实体属性表

| 实体 | 中文名称 | 主键 | 主要属性 | 属性说明 |
|---|---|---|---|---|
| sys_user | 系统用户 | id | username, password_hash, real_name, phone, email, department_id, user_type, status | 记录校内系统用户账号、身份、归属部门和启用状态 |
| sys_role | 角色 | id | role_code, role_name, role_desc, status | 定义访客、被访人、审批人员、门岗、管理员、校级管理人员等角色 |
| sys_permission | 权限 | id | permission_code, permission_name, permission_type, parent_id, route_path, api_path, status | 定义菜单权限、按钮权限和接口权限 |
| sys_user_role | 用户角色关联 | id | user_id, role_id | 表示用户与角色之间的多对多联系 |
| sys_role_permission | 角色权限关联 | id | role_id, permission_id | 表示角色与权限之间的多对多联系 |
| department | 部门 | id | parent_id, dept_code, dept_name, leader_user_id, phone, sort_order, status | 记录学校部门组织结构，支撑部门审批和统计 |
| campus_gate | 校门 | id | gate_code, gate_name, gate_location, gate_type, status, remark | 记录校门基础信息，支撑核验和通行统计 |
| visitor | 访客 | id | visitor_name, id_type, id_number, phone, gender, unit_name, visitor_type, remark | 记录校外访客身份和联系信息 |
| visitor_vehicle | 访客车辆 | id | visitor_id, plate_number, vehicle_type, vehicle_color, brand, status | 记录访客车辆信息，可被预约引用 |
| visitor_companion | 随行人员 | id | apply_id, companion_name, id_type, id_number, phone, relation_remark | 记录某次预约中的随行人员 |
| visit_apply | 访客预约 | id | apply_no, visitor_id, host_user_id, department_id, vehicle_id, visit_reason, plan_start_time, plan_end_time, apply_status, access_status | 记录访客预约申请和流程状态 |
| approval_record | 审批记录 | id | apply_id, approval_step, approver_user_id, approval_result, approval_comment, approval_time, sort_order | 记录被访人确认、部门审批等审批轨迹 |
| pass_code | 通行凭证 | id | apply_id, pass_code, qr_content, valid_from, valid_to, pass_status, used_time, verify_count | 记录审批通过后生成的通行码 |
| access_record | 出入校记录 | id | apply_id, visitor_id, pass_code_id, entry_gate_id, exit_gate_id, entry_guard_id, exit_guard_id, entry_time, exit_time, access_status | 记录访客实际入校、离校和当前访问状态 |
| blacklist | 黑名单 | id | visitor_id, id_number, phone, reason, level, start_time, end_time, status, operator_user_id | 记录限制入校或风险提示信息 |
| notice | 通知消息 | id | receiver_user_id, receiver_type, title, content, business_type, business_id, read_status, read_time | 记录预约、审批、系统事件通知 |
| operation_log | 操作日志 | id | operator_user_id, operator_name, module_name, operation_type, request_method, request_url, operation_result, ip_address, operation_time | 记录系统关键操作和异常信息 |
| dict_type | 字典类型 | id | dict_code, dict_name, description, status | 定义字典分类，如预约状态、访问状态、证件类型 |
| dict_item | 字典项 | id | dict_type_id, item_code, item_label, item_value, sort_order, status, remark | 定义字典分类下的具体取值 |
| screenshot_record | 截图记录 | id | screenshot_code, page_name, route_path, role_code, file_path, description, capture_time, status, created_by | 记录自动截图文件及其报告说明 |
| report_record | 报告记录 | id | report_code, report_name, markdown_path, word_path, generate_status, generate_time, generated_by, error_message | 记录自动生成报告的结果和错误信息 |

## 4. 属性与业务流程对应

| 业务流程 | 关键实体属性 |
|---|---|
| 预约提交 | visitor.visitor_name, visitor.id_number, visit_apply.apply_no, visit_apply.visit_reason, visit_apply.plan_start_time, visit_apply.plan_end_time |
| 黑名单检查 | blacklist.visitor_id, blacklist.id_number, blacklist.phone, blacklist.status |
| 被访人确认 | approval_record.approval_step, approval_record.approver_user_id, approval_record.approval_result |
| 部门审批 | visit_apply.department_id, approval_record.approval_comment, approval_record.approval_time |
| 通行码生成 | pass_code.pass_code, pass_code.valid_from, pass_code.valid_to, pass_code.pass_status |
| 门岗核验 | pass_code.verify_count, access_record.entry_guard_id, campus_gate.gate_name |
| 入校离校 | access_record.entry_time, access_record.exit_time, access_record.access_status |
| 统计报表 | visit_apply.apply_status, access_record.access_status, department.dept_name, campus_gate.gate_name |
| 审计追踪 | operation_log.operator_user_id, operation_log.operation_type, operation_log.operation_time |
