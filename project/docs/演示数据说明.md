# 演示数据说明

`database/demo_seed.sql` 是最终演示、自动截图和课程设计报告使用的增强数据脚本。该脚本不修改数据库表结构，使用 `CURRENT_DATE()` 和 `NOW()` 动态生成日期，保证今日、本周、本月、近七天和未来七天均有可展示数据。

## 数据规模

| 数据对象 | 规模 | 覆盖说明 |
| --- | ---: | --- |
| 部门 | 13 个 | 覆盖学院、研究生院、教务处、保卫处、后勤管理处 |
| 校门 | 7 个 | 覆盖腾飞门、北门、崇文门、西门、西南门、图书馆访客入口、创新创业园门岗 |
| 系统用户 | 31 个 | 含 2 个管理员、15 个被访教师、6 个审批人员、5 个安保、2 个校级管理人员、1 个访客账号 |
| 访客 | 140 条 | 覆盖学生家长、企业合作、后勤维修、学术交流、面试、配送、校友、外聘专家 |
| 预约申请 | 220 条 | 覆盖最近 30 天、未来 7 天、不同部门、不同校门和多种访问事由 |
| 审批记录 | 450 条 | 覆盖被访人确认、部门审批、安保复核、同意、拒绝、退回修改、待处理 |
| 通行凭证 | 150 条 | 覆盖有效、已使用、已过期、已作废 |
| 出入校记录 | 125 条 | 覆盖已入校未离校、已入校已离校、超时未离校、异常处理 |
| 黑名单 | 10 条 | 覆盖身份异常、多次超时、虚假预约、安全风险等场景 |
| 车辆 | 70 条 | 车牌号按重庆本地格式生成，并关联部分预约 |
| 随行人员 | 90 条 | 关联部分预约申请 |
| 操作日志 | 360 条 | 覆盖登录、预约、审批、入校、离校、黑名单、用户、报表查询 |
| 通知消息 | 120 条 | 覆盖预约提醒、审批提醒、超时提醒、风险提醒 |

## 业务场景覆盖

- 今日存在多个待确认、待审批、已入校、已离校、超时未离校和黑名单拦截预约。
- 近七天访客趋势、部门访客排行、校门通行统计、访问事由分布、审批状态分布、入校离校状态分布均有数据。
- 黑名单访客使用 `ACTIVE` 状态和不同风险等级，预约或门岗核验时可触发风险提示。
- 门岗核验页面可使用有效通行码 `PC100001`、`PC100002` 等进行查询。
- 系统日志页面提供 300 条以上数据，能够支撑分页和查询展示。

## 初始化方式

```bash
bash scripts/init_database.sh
```

也可以手动执行：

```bash
mysql -uroot -p < database/create_database.sql
mysql -uroot -p cqupt_visitor_system < database/schema.sql
mysql -uroot -p cqupt_visitor_system < database/demo_seed.sql
```

如果只运行普通初始化脚本，`database/seed.sql` 已同步为增强演示数据。
