# 提交说明

## 一、提交范围

建议提交 `project/` 目录下的课程设计主体内容，包括：

| 目录或文件 | 是否提交 | 说明 |
|---|---|---|
| `backend/` | 是 | Spring Boot 后端源码和配置 |
| `frontend/` | 是 | Vue 前端源码和 package 配置 |
| `database/` | 是 | 建库、建表、初始化数据、SQL 查询脚本和表设计文档 |
| `docs/` | 是 | 课程设计报告、运行说明、API 文档、测试文档、图表说明等 |
| `diagrams/` | 是 | Mermaid 图源码 |
| `screenshots/` | 是 | 自动截图 PNG 和 manifest |
| `scripts/` | 是 | 数据库初始化、启动、截图、报告生成脚本 |
| `codex-skills/` | 是 | 本项目沉淀的 Codex Skills |
| `PROJECT_PLAN.md`、`TODO.md`、`README.md` | 是 | 项目规划和说明 |
| `bug_fix_log.md` | 是 | 阶段性问题修复记录 |

## 二、不建议提交的内容

以下内容属于本地构建产物或依赖缓存，已通过 `.gitignore` 忽略：

```text
project/.tools/
project/logs/
project/backend/target/
project/frontend/node_modules/
project/frontend/dist/
__pycache__/
```

## 三、提交前检查命令

```bash
cd project
bash scripts/init_database.sh
mvn -f backend/pom.xml -DskipTests package
cd frontend && npm run build && cd ..
bash scripts/run_all.sh
python scripts/generate_report.py
```

## 四、核心交付物

| 交付物 | 路径 |
|---|---|
| 最终 README | `project/README.md` |
| 数据库脚本 | `project/database/*.sql` |
| 后端源码 | `project/backend/` |
| 前端源码 | `project/frontend/` |
| 自动截图 | `project/screenshots/*.png`、`project/screenshots/manifest.json` |
| Markdown 报告 | `project/docs/重庆邮电大学访客管理系统设计报告.md` |
| Word 报告 | `project/docs/重庆邮电大学访客管理系统设计报告.docx` |
| 测试说明 | `project/docs/系统测试说明.md`、`project/docs/测试用例.md` |
| 最终检查清单 | `project/docs/最终检查清单.md` |

## 五、Git 提交建议

提交信息可以使用：

```text
完成重庆邮电大学智慧访客预约与出入校管理系统课程设计最终整合
```

提交前建议执行：

```bash
git status --short
```

确认没有误提交 `node_modules`、`target`、`.tools`、`logs` 等本地目录。

## 六、已知说明

- 当前系统适合课程设计演示和报告提交，不建议直接用于真实校园生产环境。
- 若换机运行，需要重新安装 JDK、Maven、Node.js、MySQL、Python 依赖。
- Word 报告由脚本生成，若需进一步排版，可在生成后手动微调格式。
