# 前端页面说明

## 1. 前端技术栈

本系统前端位于 `project/frontend/`，采用 Vue 3 + Vite + Element Plus + Axios + Vue Router + ECharts 实现。页面主题统一为校园蓝 / 科技蓝风格，面向“重庆邮电大学智慧访客预约与出入校管理系统”的课程设计展示与后续自动截图。

## 3. 基础能力

| 能力 | 实现说明 |
|---|---|
| Vue Router | `src/router/index.js` 定义登录页、后台布局和全部业务页面路由。 |
| Axios | `src/utils/request.js` 统一封装请求，自动追加 `Authorization: Bearer <token>`。 |
| Token 存储 | `src/utils/auth.js` 使用 `localStorage` 保存 Token 和当前用户信息。 |
| 角色菜单 | `src/utils/menu.js` 按 `ADMIN`、`HOST`、`DEPT_APPROVER`、`GATE_GUARD`、`SCHOOL_MANAGER`、`VISITOR` 动态显示菜单。 |
| 状态标签 | `src/components/StatusTag.vue` 统一展示预约状态、访问状态、审批结果和黑名单状态。 |
| 通用表格 | `src/components/CrudTable.vue` 提供筛选、表格、分页、新增/编辑弹窗、空状态和加载状态。 |
| 图表展示 | 首页和统计报表页使用 ECharts 展示趋势、排行、校门通行和审批通过率。 |

## 4. 页面清单

| 页面 | 路由 | 主要内容 | 对应接口 |
|---|---|---|---|
| 登录页 | `/login` | 校园背景、系统标题、账号密码表单、测试账号提示、系统简介 | `POST /api/auth/login` |
| 首页统计驾驶舱 | `/dashboard` | 今日访客、当前在校、超时未离校、待审批、趋势图、部门排行、校门统计、快捷入口 | 基础 CRUD 聚合，后续可替换统计接口 |
| 访客预约申请页 | `/visit/apply` | 访客信息、证件信息、被访人、部门、访问时间、车辆、随行人员表单 | `POST /api/workflow/visit-applies` |
| 我的预约页 | `/visit/mine` | 预约列表、状态标签、详情和取消未审批预约 | `GET /api/workflow/visit-applies/my` |
| 待确认预约页 | `/approval/host` | 被访人待确认列表、确认、拒绝 | `GET /api/workflow/host/pending` |
| 部门审批页 | `/approval/department` | 部门待审批列表、审批通过、审批拒绝 | `GET /api/workflow/department/pending` |
| 门岗核验页 | `/gate/verify` | 通行码、预约编号、手机号、证件号核验；展示是否允许入校 | `POST /api/workflow/gate/verify` |
| 入校登记页 | `/access/entry` | 预约ID/编号/通行码、入校校门登记 | `POST /api/workflow/access/entry` |
| 离校登记页 | `/access/exit` | 出入记录ID、预约ID/编号、离校校门登记 | `POST /api/workflow/access/exit` |
| 当前在校访客页 | `/access/current` | 已入校未离校访客表格 | `GET /api/access-records` |
| 超时未离校访客页 | `/access/overtime` | 超时列表和标记超时操作 | `GET /api/workflow/access/overtime` |
| 黑名单管理页 | `/security/blacklist` | 黑名单查询、新增、编辑、复核、状态标签 | `/api/blacklists` |
| 访客记录查询页 | `/records` | 历史预约、审批状态、访问状态查询 | `/api/visit-applies` |
| 统计报表页 | `/reports` | 趋势、审批通过率、部门排行、校门通行统计 | 后续统计接口或现有数据聚合 |
| 用户管理页 | `/system/users` | 用户账号列表和维护表单 | `/api/sys-users` |
| 角色权限管理页 | `/system/roles` | 角色列表和维护表单 | `/api/sys-roles` |
| 部门管理页 | `/system/departments` | 部门列表和维护表单 | `/api/departments` |
| 校门管理页 | `/system/gates` | 校门列表和维护表单 | `/api/campus-gates` |
| 系统日志页 | `/system/logs` | 操作日志审计查询 | `/api/operation-logs` |

## 5. UI 美化点

1. 登录页使用 `public/cqupt-campus.jpg` 作为校园背景，叠加深蓝遮罩，突出学校场景和系统标题。
2. 全局主题使用校园蓝 `#1D4ED8`、科技青 `#06B6D4`、成功绿、警示黄和风险红，避免单一色调。
3. 后台布局包含侧边栏、顶部导航、当前角色标签、退出按钮和内容区域。
4. 业务页面统一使用页面标题、筛选卡片、数据表格、分页、状态标签和弹窗表单。
5. 首页驾驶舱提供统计卡片、趋势图、部门排行、校门通行图和快捷入口。
6. 门岗核验页使用左右分栏，左侧录入核验条件，右侧展示核验结果和状态。
7. 统计报表页使用 ECharts，适合后续自动截图插入课程设计报告。
8. 黑名单、用户、日志等管理页均提供搜索区域、操作按钮和空状态。

## 6. 启动方式

当前开发环境未检测到 `node` 和 `npm` 命令。安装 Node.js 后，在 `project/frontend/` 下执行：

```bash
npm install
npm run dev
```

默认前端地址为：`http://127.0.0.1:5173`。Vite 已配置 `/api` 代理到 `http://127.0.0.1:8080`，因此启动前请先启动 Spring Boot 后端。
