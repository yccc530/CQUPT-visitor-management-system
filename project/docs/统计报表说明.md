# 统计报表说明

本系统统计报表服务于“重庆邮电大学智慧访客预约与出入校管理系统”的首页驾驶舱、统计报表页、课程设计报告和自动截图。统计接口统一由后端 `StatisticsController` 提供，前端通过 `frontend/src/api/statistics.js` 调用 `/api/statistics/dashboard` 获取完整数据。

## 接口与页面

| 页面 | 接口 | 主要字段 | 说明 |
| --- | --- | --- | --- |
| 首页统计驾驶舱 | `/api/statistics/dashboard` | `overview`、`trend`、`departmentRank`、`gateSummary` | 展示今日访客、本周访客、当前在校、超时未离校、待审批、趋势、排行和校门通行 |
| 统计报表页 | `/api/statistics/dashboard` | 全部统计字段 | 展示趋势、部门排行、校门通行、访问事由、审批状态、出入校状态、黑名单风险和最近日志 |

## 统计项与数据来源

| 统计项 | 数据来源表 | 统计口径 |
| --- | --- | --- |
| 今日访客数 | `visit_apply` | `plan_start_time` 在当天内的预约数量 |
| 本周访客数 | `visit_apply` | `plan_start_time` 在本周一至周日内的预约数量 |
| 本月访客数 | `visit_apply` | `plan_start_time` 在当月内的预约数量 |
| 当前在校访客数 | `access_record` | `access_status = 'ENTERED'` 的出入校记录数量 |
| 超时未离校访客数 | `access_record` | `access_status = 'OVERTIME'` 的出入校记录数量 |
| 待审批预约数 | `visit_apply` | `apply_status` 属于 `PENDING_HOST`、`HOST_CONFIRMED`、`PENDING_DEPT` |
| 近七天访客趋势 | `visit_apply` | 最近 7 天每日计划来访数量 |
| 部门访客排行 | `visit_apply`、`department` | 当月按 `department_id` 聚合预约数量 |
| 校门通行统计 | `access_record`、`campus_gate` | 当月按入校门和离校门分别统计通行数量 |
| 访问事由分布 | `visit_apply` | 将 `visit_reason` 归类为学生家长、企业合作、后勤维修、学术交流、面试招聘、快递配送、校友返校、外聘专家、其他 |
| 审批状态分布 | `visit_apply` | 按 `apply_status` 统计预约状态数量 |
| 入校离校状态分布 | `visit_apply` | 按 `access_status` 统计未入校、已入校、已离校、超时未离校、异常处理数量 |
| 审批通过率 | `approval_record` | 部门审批记录中同意、拒绝/退回、待处理数量及通过率 |
| 黑名单风险数量 | `blacklist` | 按 `level` 和 `status` 统计风险数量，首页显示当前生效风险数 |
| 最近操作日志 | `operation_log` | 按 `operation_time` 倒序取最近 10 条日志 |

## 验证方式

1. 执行 `bash scripts/init_database.sh` 初始化数据库。
2. 启动后端后登录 `admin / 123456`。
3. 访问 `/api/statistics/dashboard`，检查 `overview`、`trend`、`departmentRank`、`gateSummary` 等字段是否有数据。
4. 打开前端首页和统计报表页，确认所有统计卡片和 ECharts 图表均非空。
5. 执行 `database/query_examples.sql` 中第 13 至 22 条查询，确认 SQL 与接口统计口径一致。
