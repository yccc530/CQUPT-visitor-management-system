# 用例分析

## 1. 参与者

| 参与者 | 说明 |
|---|---|
| 访客 | 校外人员，申请入校访问并查看通行凭证 |
| 被访人 | 校内人员，确认访客来访是否属实 |
| 部门审批人员 | 负责审批本部门相关访客申请 |
| 门岗安保人员 | 负责通行码核验和出入校登记 |
| 系统管理员 | 负责系统基础数据和权限管理 |
| 校级管理人员 | 负责查看全校访客记录和统计报表 |

## 3. 重点用例描述

### UC-01 提交访客预约

- 参与者：访客。
- 前置条件：访客已进入预约申请页面。
- 基本流程：
  1. 访客填写姓名、证件类型、证件号、手机号、单位、被访人、访问时间、来访事由、车辆和随行人员信息。
  2. 系统校验必填项和时间范围。
  3. 系统根据证件号和手机号检查黑名单。
  4. 未命中黑名单时，系统保存访客信息和预约申请。
  5. 系统将预约状态置为待确认，并通知被访人。
- 异常流程：命中黑名单时，系统拒绝提交或提示需人工处理。
- 后置条件：生成预约申请记录。

### UC-03 被访人确认预约

- 参与者：被访人。
- 前置条件：存在状态为待确认的预约申请。
- 基本流程：
  1. 被访人查看待确认预约。
  2. 被访人核对访客信息、来访事由和访问时间。
  3. 被访人选择确认或拒绝，并填写意见。
  4. 系统写入审批记录并更新预约状态。
- 异常流程：预约已取消或已处理时不能重复确认。
- 后置条件：预约进入待部门审批或被访人已拒绝状态。

### UC-04 部门审批预约

- 参与者：部门审批人员。
- 前置条件：预约已通过被访人确认。
- 基本流程：
  1. 审批人员查看本部门待审批预约。
  2. 审批人员查看访客、被访人、访问时间和来访事由。
  3. 审批人员选择通过或拒绝，并填写审批意见。
  4. 系统记录审批结果。
  5. 审批通过时系统生成通行凭证。
- 异常流程：非本部门审批人员不能审批该预约。
- 后置条件：预约进入审批通过或审批拒绝状态。

### UC-06 门岗核验

- 参与者：门岗安保人员。
- 前置条件：访客持有通行凭证并到达校门。
- 基本流程：
  1. 门岗安保人员扫描或输入通行码。
  2. 系统查询通行凭证、预约状态和黑名单信息。
  3. 系统判断通行凭证是否有效、预约是否审批通过、访客是否被限制入校。
  4. 核验通过时允许办理入校登记。
  5. 核验失败时返回失败原因。
- 异常流程：通行码过期、预约未审批通过、访客在黑名单中均不能入校。
- 后置条件：形成核验结果并记录操作日志。

### UC-07 入校登记

- 参与者：门岗安保人员。
- 前置条件：门岗核验通过，访客尚未入校。
- 基本流程：
  1. 门岗安保人员选择入校校门并确认入校。
  2. 系统写入入校时间、校门和经办人。
  3. 系统将访问状态更新为已入校。
- 异常流程：已入校记录不能重复入校。
- 后置条件：生成或更新出入校记录。

### UC-08 离校登记

- 参与者：门岗安保人员。
- 前置条件：访客已入校且尚未离校。
- 基本流程：
  1. 门岗安保人员查询当前在校访客。
  2. 选择访客并确认离校。
  3. 系统写入离校时间、离校校门和经办人。
  4. 系统将访问状态更新为已离校。
- 异常流程：已离校记录不能重复离校。
- 后置条件：访问记录归档，可用于查询统计。
